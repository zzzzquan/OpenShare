# OpenShare 核心实体关系设计

本文档只定义阶段二 `2.1` 所需的核心实体与关系边界，不在本阶段提前定稿全部字段、索引和状态机。

## 设计原则

- 数据库存放元数据，磁盘文件内容由本地文件系统负责。
- 状态流转仍由 `service` 层统一控制，不在模型层直接承载复杂业务逻辑。
- 关系设计优先保证上传、审核、浏览、搜索、举报、公告这几条主链路能够闭环。
- `tag_submissions` 功能保留，因为技术文档明确要求“用户可提交 Tag（需管理员审核）”。

## 核心实体

### 1. `admins`

管理后台操作主体。

关系：

- 一个 `admin` 可以审核多个 `submissions`
- 一个 `admin` 可以处理多个 `reports`
- 一个 `admin` 可以发布多个 `announcements`
- 一个 `admin` 可以产生多条 `operation_logs`

### 2. `folders`

资源目录节点，支持树形结构。

关系：

- 一个 `folder` 可以属于一个父目录
- 一个 `folder` 可以拥有多个子目录
- 一个 `folder` 可以包含多个 `files`
- 一个 `folder` 可以绑定多个 `tags`
- 一个 `folder` 可以被多个 `reports` 指向

### 3. `files`

资源元数据主体，对应本地文件系统中的实际文件。

关系：

- 一个 `file` 至多属于一个 `folder`
- 一个 `file` 至多关联一个 `submission`
- 一个 `file` 可以绑定多个 `tags`
- 一个 `file` 可以被多个 `reports` 指向

### 4. `submissions`

用户上传审核记录，承接 `staging -> 审核 -> repository` 的流程。

关系：

- 一个 `submission` 至多生成一个最终 `file`
- 一个 `submission` 至多由一个 `admin` 审核

### 5. `tags`

独立标签实体。

关系：

- 一个 `tag` 可以关联多个 `files`
- 一个 `tag` 可以关联多个 `folders`
- 一个 `tag` 可以对应多个 `tag_submissions` 记录

### 6. `file_tags`

`files` 与 `tags` 的关联表。

关系：

- 每条记录只指向一个 `file`
- 每条记录只指向一个 `tag`

### 7. `folder_tags`

`folders` 与 `tags` 的关联表。

关系：

- 每条记录只指向一个 `folder`
- 每条记录只指向一个 `tag`

### 8. `reports`

用户举报记录。

关系：

- 每条 `report` 必须且只能指向一个资源目标：`file` 或 `folder`
- 每条 `report` 至多由一个 `admin` 处理

### 9. `announcements`

首页公告实体。

关系：

- 每条 `announcement` 由一个 `admin` 创建

### 10. `operation_logs`

后台审计日志实体。

关系：

- 每条 `operation_log` 可关联一个后台操作人 `admin`
- 具体操作目标使用 `target_type + target_id` 一类字段在后续 `2.2` 定稿

### 11. `tag_submissions`

候选 Tag 提交记录。

关系：

- 每条 `tag_submission` 可在审核通过后关联一个正式 `tag`
- 每条 `tag_submission` 至多由一个 `admin` 审核

## 关系总览

- `admins` 1 -> n `submissions`
- `admins` 1 -> n `reports`
- `admins` 1 -> n `announcements`
- `admins` 1 -> n `operation_logs`
- `folders` 1 -> n `folders`（自关联）
- `folders` 1 -> n `files`
- `files` 1 -> 0..n `reports`
- `folders` 1 -> 0..n `reports`
- `submissions` 1 -> 0..1 `files`
- `files` n <-> n `tags` via `file_tags`
- `folders` n <-> n `tags` via `folder_tags`
- `tags` 1 -> n `tag_submissions`
