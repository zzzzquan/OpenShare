# OpenShare

OpenShare 是一个面向校园网、实验室和组织内部网络的轻量级、自托管资料共享平台，用于支持学习资料与组织内部资料的统一沉淀、检索、分发和治理。

项目聚焦三个目标：

- 让普通用户无需注册即可完成浏览、搜索、上传和下载
- 让管理员能够围绕审核、公告、举报、Tag 和资源维护开展日常治理
- 让系统保持轻量、低依赖、易部署，适合中小规模内网场景落地

## 适用场景

OpenShare 主要适用于以下环境：

- 校园网资料共享
- 实验室内部资料沉淀
- 研究组服务器文件归档
- 组织内部学习与知识共享

它不是面向开放公网的大型内容平台，而是一个强调可控、自部署、低维护成本的内网资料库系统。

## 角色与使用方式

系统包含三类角色：

- `Guest`：无需注册即可浏览、搜索、下载和上传资料
- `Admin`：负责审核资料、处理举报、维护内容质量
- `Super Admin`：负责管理员管理、系统策略配置和全局运营治理

认证策略如下：

- 用户侧默认免登录
- 管理端采用 `Session + Cookie`
- 管理员密码使用 `bcrypt` 存储

## 核心能力

### 资料浏览与下载

- 展示公开资料列表与详情
- 支持单文件下载
- 支持批量下载与打包下载
- 记录资料下载次数

### 搜索与检索

- 支持关键词搜索
- 支持模糊匹配
- 支持 Tag 过滤与组合搜索
- 支持按目录范围限定搜索
- 搜索对象覆盖文件与文件夹
- 搜索结果按相关度和下载热度综合排序

### 投稿与审核

- 访客可直接上传资料
- 上传内容先进入暂存区，再进入审核流程
- 用户可通过回执码查询投稿状态
- 管理员可执行通过或驳回，并填写驳回原因

### 资料治理

- 支持公告发布与展示
- 支持 Tag 创建、编辑、合并和审核
- 支持文件与文件夹举报
- 支持资料编辑、下架和删除
- 支持管理员操作日志记录

### 本地资料导入

- 支持从服务器本地目录导入已有文件
- 系统负责识别目录结构、建立元数据并纳入统一检索

## 系统设计原则

OpenShare 采用“本地文件存储 + 数据库存储元数据”的模式：

- 文件内容保存在本地文件系统
- 文件、文件夹、Tag、投稿、举报、公告等业务数据存储在数据库中
- 上传文件先写入 `staging`，审核通过后再进入正式资料目录
- 删除操作优先进入 `trash`，便于治理和回收

这套设计兼顾了部署简单性、数据可控性和后续治理能力。

## 搜索与 Tag 策略

系统搜索以 SQLite `FTS5` 为核心，结合业务规则提供轻量但可用的检索体验：

- 全文索引字段包括文件名、文件夹名和 Tag
- 支持前缀匹配与辅助模糊搜索
- 搜索忽略大小写
- 文件可继承父文件夹 Tag 参与搜索
- Tag 为独立实体，文件和文件夹都可绑定
- Tag 名称全局唯一，且忽略大小写判重

## 文件预览策略

系统仅对稳定、低成本格式提供在线预览：

- 支持：PDF、图片、纯文本
- 不支持：Office 文档、音视频等复杂格式

不支持预览的文件以下载为主，避免引入额外转换链路和部署复杂度。

## 安全与治理策略

为适应内网共享场景，系统默认内置以下治理能力：

- 上传与搜索接口支持限流策略
- 记录上传者 IP 与管理员操作 IP
- 管理端基于权限模型控制功能访问范围
- 公开侧不展示上传者身份信息

## 设计取向

OpenShare 的设计取向不是“大而全”，而是“围绕资料共享主链路，把上传、检索、审核、治理做完整”。它强调：

- 低门槛使用
- 低成本部署
- 清晰的权限边界
- 可持续的资料治理

如果你的目标是在一个可控的内网环境中快速搭建资料共享系统，OpenShare 是为此场景设计的。
